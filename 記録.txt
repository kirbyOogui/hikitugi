========================================
引継書システム 開発記録
========================================

■ 2026-07-03: 初回実装

【依頼内容】
店舗のホワイトボード・口頭引継ぎを置き換える「引継書システム」の新規開発。
- Backend: FastAPI / Frontend: HTML・CSS・JS / DB: PostgreSQL(SQLAlchemy)
- 画像: Cloudinary / Deploy: Render / 環境変数: .env
- 画面はホーム・設定の2画面のみ、追加編集はモーダル/ボトムシート
- Repository/CQRS等は使わずシンプルな構成にする指示

【設計フェーズで確認した事項（ユーザー回答）】
1. カテゴリ削除時、紐づく引継ぎがあれば削除禁止（未対応/対応済み問わず、DB制約と一致させるため全件チェックに統一）
2. カテゴリ並び替えはドラッグ&ドロップ（SortableJSをCDNで導入）
3. 引継ぎの写真は複数枚添付可能（handover_photosテーブルを分離）
4. 認証なし（店舗内共有端末での利用前提）
5. 日付・時刻表示はAsia/Tokyo固定

【実装したもの】
- DB: categories / handovers / handover_photos / soldout_items / garbage_status の5テーブル
  - マイグレーションツール(Alembic等)は個人開発規模のため未導入。起動時にBase.metadata.create_all()でテーブル作成。
  - garbage_statusはid=1固定のシングルトン行として運用。
- API: /api/categories, /api/handovers, /api/soldout, /api/garbage （役割ごとにrouterファイルを分割）
  - 引継ぎ追加はmultipart/form-dataで受け取り、バックエンド側でCloudinaryへアップロードしてURL+public_idをDB保存
  - 引継ぎ完全削除時はCloudinary側の画像もあわせて削除（失敗時はログのみ、DB削除は継続）
- フロント: index.html(ホーム) / settings.html(設定)、ビルド不要の素のJS
  - FABから「引継ぎ追加」「売切商品追加」をボトムシートで選択
  - ゴミ庫トグルは押下後即座にPUTし、レスポンス確定後に表示反映
  - 設定画面のカテゴリ並び替えはSortableJS(CDN)のドラッグ&ドロップ

【発生した問題と対応】
- Windows/Git Bashのcurlで日本語を含む -F / -d 引数を渡すと文字化けする事象を確認。
  → Pythonのurllibで直接UTF-8バイト列を送信して検証したところ、サーバー側の保存・応答は正しいUTF-8であることを確認。curl側のシェル引数エンコーディングの問題であり、アプリ側のバグではないと判断。
- settings.jsで、カテゴリ編集時にnameを文字列テンプレートでouterHTMLに埋め込んでいた箇所がXSSの余地があったため、DOM API（createElement/replaceWith）で安全に組み立てる実装に修正。
- Playwrightでのスクリーンショット確認時、ボトムシート表示直後（CSSアニメーション"sheet-up"の途中、0.15秒)を撮影すると背後の要素と二重写りして見えることを確認。アニメーション完了を待ってから再撮影し、正常表示であることを確認済み（実装上の問題ではなく撮影タイミングの問題）。

【動作確認】
- ローカルでSQLite(一時利用、動作確認専用)+ダミーCloudinary値の.envを使い、uvicornを起動して全APIエンドポイントをテスト
  - カテゴリ一覧・追加・編集・並び替え・削除制限(400)
  - 引継ぎ一覧・追加(写真なし/multipart)・対応済み・再登録・完全削除
  - 売切商品追加・一覧・削除
  - ゴミ庫状態取得・更新
- Playwright(システムEdge, headless)でホーム画面・設定画面をブラウザ描画し、FABメニュー・引継ぎ追加ボトムシート・登録後反映・設定画面のカテゴリ一覧/対応済み一覧の見た目を目視確認。コンソールエラーはfavicon.ico の404のみ（機能に影響なし）。
- 確認用に作成した.env・test_local.db・server.logは検証後に削除済み。本番用の実DB・Cloudinary設定は未実施（ユーザーが別途.envを用意する想定）。

■ 2026-07-03: アンダーバー機能の解釈修正

【依頼内容】
前回実装した「日付に自動で下線」機能について、ユーザーから「更新日ではなく、文章内に入っている日付のこと」との指摘。引継ぎ本文中に書かれた日付表記（例: 7/5, 7月5日など）を自動検出して下線表示してほしい、という趣旨に修正。

【実装内容】
- style.cssの.handover-card__updatedに付けていたtext-decoration:underlineを削除（誤った解釈だったため撤回）。
- static/js/utils.js を新規作成。home.js/settings.js両方から使う共通ユーティリティとして以下を実装:
  - DATE_PATTERN: 「7/5」「2026/7/10」「7月5日」「2026年7月5日」「7月5日(土)」等にマッチする正規表現
  - buildTextWithHighlightedDates(text): 本文テキストをDocumentFragmentに変換し、日付部分だけ<span class="date-highlight">で囲む。textContent/createTextNodeベースで組み立てるため、ユーザー入力をinnerHTMLに流し込む形にはならずXSSの心配はない。
- CSSに.date-highlight（青の下線+太字で強調）を追加。
- home.jsの引継ぎカード本文、settings.jsの対応済み一覧本文の両方をtextContent直書きからbuildTextWithHighlightedDates()経由に変更。
- index.html / settings.html の両方にutils.jsの<script>読み込みを追加（api.jsの後、home.js/settings.jsの前）。

【動作確認】
- Node上でDATE_PATTERNの正規表現を単体テストし、「7/5」「7月5日(土)」「2026年7月10日」「2026/7/10」を正しく検出、日付を含まない文章・「3/4」のような数字/数字表記も期待通りマッチすることを確認。

- 同じ引継ぎを対応済みにして設定画面の対応済み一覧に移動させ、そちらでも同様に日付が強調表示されることを確認。
